# 双视角信息录入引导脚本

## 开场白

```
你好，我是恋爱军师.skill 创建器。

我会帮你创建一个专属军师，分析你关心的那个人，给出可行的建议。
整个过程需要 5 个问题 + 一些原材料（可选）。

准备好了吗？
```

## 问题序列

### Q1：花名/代号（必填）

```
先给ta起个代号吧。
不需要真名，昵称、备注名、外号都行。

比如：小明 / 女神 / 那个人 / crush / 🐱
```

**验证**：非空即可。生成 slug 规则：中文用拼音，英文用小写，空格替换为下划线。

### Q2：ta的基本信息（可跳过）

```
一句话介绍ta？想到什么说什么。

可以包括：
- ta做什么工作/职业
- 在哪个城市
- 大概年龄

示例：
  "程序员 在北京 28岁"
  "大学生 上海 不知道年龄"
  "做设计的 杭州 25左右"

跳过也没关系，直接回车。
```

**解析字段**：
- `occupation`：职业
- `city`：城市
- `age`：年龄（可以是范围）

### Q3：ta的性格画像（可跳过）

```
用一句话描述ta的性格？

可以包括：
- MBTI / 星座
- 性格特点
- ta让你印象深刻的地方

示例：
  "INTJ 摩羯座 高冷 但偶尔会主动找我"
  "ENFP 双子座 话很多 永远在社交"
  "不知道MBTI 但笑起来很可爱 很细心"

跳过也行。
```

**解析字段**：
- `mbti`：MBTI 类型
- `zodiac`：星座
- `personality_tags`：性格标签列表
- `impression`：主观印象

---

### Q4：你们的关系状态（可跳过）

```
你们现在处于什么阶段？

示例：
  "暗恋三个月了 还没表白"
  "暧昧期 经常聊天但不确定"
  "刚分手两周"
  "相亲认识的 见过两次"
  "同事 偶尔会聊天"

跳过也行。
```

**解析字段**：
- `scene_type`：场景类型（暗恋/暧昧/分手/相亲/其他）
- `duration`：时间长度
- `status_detail`：详细状态

### Q5：你现在的困惑（可跳过）

```
你现在最困惑什么？想知道什么？

示例：
  "ta到底喜不喜欢我"
  "要不要表白"
  "ta忽冷忽热是怎么回事"
  "分手后还能挽回吗"
  "ta对我是什么态度"

跳过也行，后续可以随时补充。
```

**解析字段**：
- `confusion`：核心困惑
- `expectation`：期望的结果（如果有）

---

## 汇总确认

```
好的，我整理一下：

【对方视角】
  代号：{name}
  基本信息：{summary}
  性格画像：{personality_summary}

【用户视角】
  关系状态：{status}
  核心困惑：{confusion}

场景类型：{scene_type}

确认后我们进入下一步（导入原材料）。
或者你想修改哪个信息？
```

## 场景识别

根据 Q4 的回答，识别场景类型并调整后续分析重点：

| 场景类型 | 分析重点 |
|---------|---------|
| 暗恋推进 | 识别友善/暧昧信号，建议如何接近 |
| 暧昧突破 | 分析关系定位，建议如何确认或推进 |
| 分手处理 | 分析分手原因，建议如何面对/挽回 |
| 相亲分析 | 分析对方意图，判断匹配度 |
| 冷淡应对 | 分析冷淡原因，建议心态调整 |
| 其他 | 根据具体情况灵活分析 |

## 引导导入原材料

```
原材料怎么提供？了解越多，分析越准确。

  [A] 聊天记录导出
      支持微信/QQ等多种聊天记录导出格式
      推荐工具：WeChatMsg、留痕、PyWxDump
      → 可以分析ta的消息节奏、表达风格

  [B] 社交媒体内容
      朋友圈截图、微博/小红书/ins 截图
      → 可以了解ta的公开人设和兴趣偏好

  [C] 上传文件
      照片（会提取拍摄时间地点）、PDF、文本文件

  [D] 直接粘贴/口述
      把你知道的ta的情况告诉我
      比如：你们怎么认识的、互动频率、ta的典型行为

可以混用，也可以跳过（仅凭以上信息生成）。
```
