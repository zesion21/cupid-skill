# 增量合并逻辑

## 任务

当用户提供新的原材料或信息时，将增量内容合并到现有的 profile.md 和 context.md 中。

**原则**：不覆盖已有结论，追加或修正。

---

## 合并流程

### 1. 分析增量内容

识别新内容属于哪个维度：

**Target Profile 增量**：
- 新的聊天记录 → 更新消息节奏、话题偏好
- 新的社交内容 → 更新性格推断、兴趣偏好
- 新的互动事件 → 更新关系信号、关系定位

**User Context 增量**：
- 新的倾诉内容 → 更新困惑点、主观感受
- 新的尝试行动 → 追加已尝试行动
- 新的时间节点 → 更新时间线

### 2. 与现有内容对比

判断增量内容与现有内容的关系：

| 关系类型 | 处理方式 |
|---------|---------|
| 新增信息 | 追加到对应部分 |
| 补充信息 | 扩充现有结论 |
| 修正信息 | 更新现有结论，标注修改时间 |
| 冲突信息 | 保留两者，标注"用户反馈修正" |

### 3. 合并策略

**不覆盖原则**：

```
❌ 不好的做法：直接覆盖
原有：ta回复速度较快
新增：ta最近回复变慢
→ 直接改成：ta回复速度较慢

✅ 好的做法：追加变化
原有：ta回复速度较快（观察期：1月）
新增：ta最近回复变慢（观察期：2月）
→ 合并为：
  - 1月：回复速度较快
  - 2月：回复速度变慢（可能的态度变化信号）
```

## 合并模板

### profile.md 合并格式

```markdown
## 消息节奏

### 历史观察
- {时间段}：{观察结果}

### 最新观察（更新时间：{时间}）
- {时间段}：{新观察结果}

### 变化分析
{如有变化，分析趋势}
```

### context.md 合并格式

```markdown
## 时间线

- {时间点}：{事件}
- {时间点}：{事件}（新增）

## 已尝试行动

- {时间}：{行动} → {结果}
- {时间}：{行动} → {结果}（新增）

## 倾诉记录摘要

### 会话 {id}（{时间}）
{摘要}

### 会话 {id}（{时间}）（新增）
{摘要}
```

## 版本管理

每次合并时：

1. 存档当前版本
```bash
python3 ${CLAUDE_SKILL_DIR}/tools/version_manager.py --action backup --slug {slug} --base-dir ./targets
```

2. 更新内容

3. 更新 meta.json
```json
{
  "version": "v{n}",
  "updated_at": "{ISO时间}",
  "changelog": [
    "{本次更新内容}"
  ]
}
```

## 冲突处理

当新信息与旧结论冲突时：

**可能的冲突**：
- 用户纠正：说之前的判断不对
- 行为变化：对方行为确实发生了变化
- 观察偏差：之前观察不准确

**处理方式**：

```markdown
## 消息节奏

### 历史判断（已修正）
- 原判断：ta回复较快（基于1月观察）
- 用户反馈：其实ta平时回复不快，那次可能是因为心情好
- 修正：ta回复速度不稳定，受情绪影响

### 当前判断
- ta回复速度不稳定，快慢取决于情绪状态
```

---

## 输出要求

合并后输出：
1. 更新的 profile.md（追加/修正）
2. 更新的 context.md（追加/修正）
3. 更新的 SKILL.md（合并最新内容）
4. 更新的 meta.json（版本+时间）
