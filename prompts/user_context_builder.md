# 用户背景构建器

## 任务

从用户填写的信息和倾诉内容中，构建 User Context（用户背景），记录用户的主观视角。

**原则**：记录用户的主观感受和认知，不评判对错，作为军师分析的重要补充视角。

---

## 构建维度

### 1. 关系时间线

梳理用户和对方的关系发展时间线：

```
时间线格式：
- {时间点}：{事件描述}
- {时间点}：{事件描述}
...

示例：
- 2024年9月：在公司认识
- 2024年10月：开始有工作上的接触
- 2024年11月：加了微信，偶尔聊天
- 2024年12月：聊得越来越频繁
- 2025年1月：ta主动约吃饭
- 2025年2月：现在，暗恋状态
```

**提取来源**：
- 用户填写的"认识多久"
- 导入的聊天记录中的时间信息
- 用户口述的时间节点

### 2. 关系状态定义

明确当前的关系状态：

| 状态类型 | 定义 |
|---------|------|
| 暗恋 | 单方面喜欢，对方可能不知道 |
| 暧昧 | 双方有互动，但关系不明确 |
| 追求中 | 正在主动追求，对方知道意图 |
| 刚接触 | 刚认识不久，互动有限 |
| 分手后 | 曾经在一起，现已分开 |
| 相亲接触 | 通过相亲认识，在接触阶段 |
| 其他 | 需要具体描述 |

```
输出格式：
关系状态：[类型]
持续时间：[多久]
状态描述：[用户的主观描述]
```

### 3. 用户困惑点

记录用户的核心困惑和想知道的问题：

```
困惑分类：
- 态度困惑：ta对我是什么态度？ta喜不喜欢我？
- 行动困惑：要不要表白？要不要主动？
- 解读困惑：ta的行为是什么意思？忽冷忽热怎么回事？
- 选择困惑：要不要继续？要不要放弃？
- 情感困惑：我是不是太执着了？我该怎么办？

输出格式：
核心困惑：[问题描述]
困惑类型：[类型]
相关困惑：[其他相关问题]
```

### 4. 用户期望

记录用户期望的结果：

```
期望分类：
- 希望推进：想在一起，想表白成功
- 希望确认：想知道ta的态度，想确认关系
- 希望理解：想理解ta的行为，想知道原因
- 希望放下：想走出这段感情，想恢复平静
- 希望挽回：想修复关系，想重新在一起
- 不确定：不知道自己想要什么

输出格式：
期望结果：[期望描述]
期望类型：[类型]
```

### 5. 已尝试的行动

记录用户已经尝试过的行动：

```
行动记录：
- {时间}：{行动} → {结果/反馈}
- {时间}：{行动} → {结果/反馈}
...

示例：
- 上周：主动找ta聊天 → 聊了半小时，ta态度友善
- 前天：发了一张照片分享 → ta回复了"挺好的"
- 昨天：约ta周末吃饭 → ta说"看情况"
```

### 6. 用户主观感受

记录用户的主观感受和认知：

```
感受记录：
- 对ta的印象：{用户描述}
- 互动感受：{用户描述}
- 焦虑点：{用户描述}
- 希望点：{用户描述}

示例：
- 对ta的印象：ta很高冷，但偶尔会主动找我，让人猜不透
- 互动感受：聊天很开心，但总是我主动，有点累
- 焦虑点：不知道ta是不是只是礼貌回复
- 希望点：ta主动约吃饭那次，让我觉得有希望
```

---

## 动态更新机制

User Context 不是静态的，每次用户倾诉后需要更新：

```
更新触发：
- 用户倾诉新情况
- 用户反馈困惑变化
- 用户尝试新行动
- 用户期望改变

更新内容：
- 时间线新增节点
- 困惑点更新
- 已尝试行动追加
- 主观感受更新
```

---

## 输出格式

生成 User Context 文档结构：

```markdown
# 用户背景

## 关系时间线
{时间线}

## 关系状态
{状态定义}

## 核心困惑
{困惑描述}

## 期望结果
{期望描述}

## 已尝试行动
{行动列表}

## 主观感受
{感受记录}

## 倾诉记录摘要
{历次倾诉的关键信息摘要}
```

---

## 注意事项

1. **主观性记录**：记录用户的感受，不评判对错
2. **动态更新**：每次倾诉后更新，反映最新状态
3. **隐私保护**：敏感信息可选择性记录
4. **情感健康**：如果用户表现出不健康执念，记录并提醒
